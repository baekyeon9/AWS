# AWS Auto Scaling

**필요한 만큼 컴퓨터를 자동으로 생성/제거 해줌**

**Elastic Load Balancer를 이용하면 많은   client가 접속 했을 때, 여러대의 EC2를 이용하여 분산하여 작업을 처리할 수 있다.**



**클라우드 컴퓨팅을 사용하는 중요한 이유 = 유연함**

언제든지 컴퓨터를 켜서 더 많은 데이터를 처리할 수 있고,  언제든지 컴퓨터를 꺼서 비용을 절약할 수 있다. 가해지는 부하에  때라서 자동으로 컴퓨터가 생성되고 사라질 수 있다면?

이 작업을 사람이 직접하지 않고  기계가 자동으로 해줄 수 없을까? 하는 고민에서 나온 서비스가  오토 스케일링이다.  

이 수업에서는  Elastic Load Balancing에 연결된 컴퓨터가  CPU 점유율이 50% 이상인 상태가 3분 이상 지속 되었을 때 켜지고,  CPU 점유율이 30% 이하인 상태가 15분 이상 지속 되었을 때 꺼지는  시나리오를 구현해본다.
